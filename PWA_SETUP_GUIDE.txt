PWA Setup Guide - Tranche Calculator

===

WHAT YOU NOW HAVE (PWA-Ready)

Your app is now a Progressive Web App (PWA). This means:

✓ Installable on any device (phone, tablet, desktop)
✓ Works offline (calculator still functions without internet)
✓ Runs in standalone mode (looks like a native app, not a browser)
✓ Can be added to home screen / app drawer
✓ Fast loading (files cached locally)
✓ No app store needed (users install directly from web)

===

NEW FILES ADDED

1. manifest.json
   - App metadata (name, description, icons, colors)
   - Tells the device "this is an app"
   - Icons used: $ symbol in blue, automatically generated

2. service-worker.js
   - Handles offline caching
   - Caches all app files on first visit
   - Works offline after cached

3. Updated index.html
   - Links to manifest.json
   - Registers service worker
   - Sets theme color and favicon

===

HOW TO DEPLOY (Same as before, but better)

1. Upload to Cloudflare Pages:
   - index.html
   - styles.css
   - script.js
   - manifest.json (NEW)
   - service-worker.js (NEW)

2. All files go in root directory (same level)

3. That's it. Cloudflare handles the rest.

===

HOW USERS INSTALL

Desktop (Chrome / Edge):
1. Visit your site URL
2. Click address bar → "Install" button (or menu → "Install app")
3. App opens in standalone window

Mobile (iOS 16.4+):
1. Visit your site in Safari
2. Tap Share → "Add to Home Screen"
3. App appears on home screen, works offline

Mobile (Android):
1. Visit your site in Chrome
2. Menu (3 dots) → "Install app" or "Add to Home Screen"
3. App appears in app drawer, works offline

===

TESTING LOCALLY (Before uploading)

If testing on your computer:

1. Run a local server:
   python3 -m http.server 8000
   (or npx http-server)

2. Visit http://localhost:8000

3. Open DevTools (F12 → Application tab)
   - You'll see service-worker.js listed
   - Manifest.json shows app details
   - Cache storage shows cached files

4. Go offline (DevTools → Network tab → "Offline")
   - App still works
   - Calculator functions normally
   - Proves caching is working

===

WHAT GETS CACHED OFFLINE

✓ index.html (main page)
✓ styles.css (all styling)
✓ script.js (all calculator logic)
✓ manifest.json (app metadata)
✓ service-worker.js (itself)

✗ NOT cached: External links, API calls, images from URLs
   (But your app doesn't use any of those, so you're fully offline-capable)

ICON & BRANDING

Current setup:
- App icon: Blue $ symbol (auto-generated SVG)
- Theme color: #185FA5 (blue)
- App name: "How much should I spend?"
- Short name: "Tranche Calculator" (appears on home screen)

If you want custom icons later:
- Create 192x192 and 512x512 PNG files
- Replace the SVG data URLs in manifest.json with file paths
- Example: "src": "icon-192.png"

===

METRICS & MONITORING

After uploading to Cloudflare:

1. Check installation stats:
   - Cloudflare Analytics → Traffic
   - Look for "Web app manifest" requests (means people are installing)

2. Check service worker activity:
   - Browser DevTools → Application → Service Workers
   - Should show "activated and running"

3. Offline usage:
   - No built-in tracker, but you can manually test:
     - Go offline, use calculator, numbers should persist

===

FUTURE UPDATES

If you want to push updates:

1. Edit files (styles.css, script.js, etc.)
2. Upload new versions to Cloudflare
3. Service worker auto-updates on next user visit
4. Users get latest version without reinstalling

(Optional: Increment CACHE_NAME in service-worker.js to force refresh)

Example: Change "v1" to "v2"
const CACHE_NAME = 'bnbs-tranche-v2';

===

IMPORTANT NOTES

- PWA works best on HTTPS (Cloudflare provides this automatically)
- Service worker won't register on localhost without HTTPS or 127.0.0.1
- Offline experience is calculator only (no new essays load, but that's fine)
- Users can still access calculator on airplane mode, in tunnels, anywhere
